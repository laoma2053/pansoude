# 排序算法

## 评分公式

```
总得分 = 插件等级得分 + 时间新鲜度得分 + 关键词匹配得分
```

三个维度的权重分配：

| 维度 | 最高得分 | 占比 |
|------|---------|------|
| 插件等级 | 1000 | ~52% |
| 时间新鲜度 | 500 | ~26% |
| 关键词匹配 | 420 | ~22% |

## 时间新鲜度得分

| 发布时间 | 得分 |
|---------|------|
| ≤ 1 天 | 500 |
| ≤ 3 天 | 400 |
| ≤ 1 周 | 300 |
| ≤ 1 月 | 200 |
| ≤ 3 月 | 100 |
| ≤ 1 年 | 50 |
| > 1 年 | 20 |
| 无日期 | 0 |

## 关键词匹配得分

结果标题中包含以下关键词时获得加分：

| 关键词 | 得分 |
|--------|------|
| 合集 | 420 |
| 系列 | 350 |
| 全 | 280 |
| 完 | 210 |
| 最新 | 140 |
| 附 | 70 |
| 无匹配 | 0 |

## 实际排序示例

| 场景 | 插件等级 | 时间 | 关键词 | 总分 | 排名 |
|------|---------|------|--------|------|------|
| 等级1 + 1天内 + 合集 | 1000 | 500 | 420 | **1920** | 1 |
| 等级1 + 1天内 + 系列 | 1000 | 500 | 350 | **1850** | 2 |
| 等级1 + 1天内 + 无关键词 | 1000 | 500 | 0 | **1500** | 3 |
| 等级2 + 1天内 + 合集 | 500 | 500 | 420 | **1420** | 4 |
| 等级3 + 1天内 + 合集 | 0 | 500 | 420 | **920** | 5 |

## 排序稳定性

`mergeResultsByType` 按原始排序顺序遍历结果，保证 `merged_by_type` 中各类型的链接顺序与整体排序一致，避免 Go map 遍历随机性导致的顺序不稳定问题。
